# Contributing to FlutterClaw

Welcome! This document gives guidelines for contributing to FlutterClaw.

## Quick Links

- **GitHub:** https://github.com/flutterclaw/flutterclaw
- **README:** [README.md](README.md) — setup, building, and architecture
- **Contact:** contact@flutterclaw.ai

## How to Contribute

1. **Bugs & small fixes** → Open a PR!
2. **New features / architecture** → Start a [GitHub Discussion](https://github.com/flutterclaw/flutterclaw/discussions) or open an issue first
3. **Questions** → Open an [issue](https://github.com/flutterclaw/flutterclaw/issues)

## Before You PR

- Test locally with your FlutterClaw build (device or emulator)
- Run: `flutter pub get && flutter analyze && flutter test`
- Keep PRs focused (one thing per PR; do not mix unrelated concerns)
- Describe what & why in the PR
- **Include screenshots** — one showing the problem/before, one showing the fix/after (for UI or visual changes)
- Use American English spelling and grammar in code, comments, docs, and UI strings

## AI-Assisted PRs

Built with Codex, Claude, or other AI tools? Please:

- Mark as AI-assisted in the PR title or description
- Note the degree of testing (untested / lightly tested / fully tested)
- Include prompts or session logs if possible
- Confirm you understand what the code does

AI PRs are welcome; we just want transparency so reviewers know what to look for.

## Current Focus & Roadmap

See the **Roadmap** section in [README.md](README.md) for current priorities and planned work.

## Report a Vulnerability

Report security issues to **security@flutterclaw.ai**. Include a clear description, steps to reproduce, impact, and remediation advice if possible.
